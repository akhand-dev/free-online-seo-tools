What is Base64 Encoding? A Complete Guide

Published on May 12, 2026 • 4 min read
[Ad Space]

If you've ever seen a string of seemingly random characters like SGVsbG8gV29ybGQ= in a URL, email header, or API response, you've encountered Base64 encoding. Let's break down what it is and why it's everywhere.

Base64 in Simple Terms

Base64 is a method of converting binary data (like images, files, or any bytes) into a string of printable ASCII characters. It uses a set of 64 characters: A-Z, a-z, 0-9, +, and /, with = used for padding.

Why Does Base64 Exist?

Many systems (email, JSON, XML, URLs) were designed to handle only text. When you need to transmit binary data through these text-only channels, you need to convert it to text first. That's exactly what Base64 does.

How It Works

  1. Take the input data and convert each character to its 8-bit binary representation
  2. Group the bits into chunks of 6 (since 2^6 = 64)
  3. Map each 6-bit chunk to one of the 64 characters
  4. Add = padding if the input isn't divisible by 3 bytes

Example

"Hello" in Base64 becomes SGVsbG8=

[Ad Space]

Common Use Cases

  • Email attachments: MIME encoding uses Base64 to embed binary files in email messages
  • Data URIs: Embedding small images directly in HTML/CSS as data:image/png;base64,...
  • API authentication: HTTP Basic Auth sends credentials as Base64-encoded strings
  • Storing binary in JSON: Since JSON only supports text, binary data must be Base64-encoded

Is Base64 Encryption?

No! This is a common misconception. Base64 is encoding, not encryption. Anyone can decode a Base64 string instantly. It provides zero security. Never use it to "protect" sensitive data like passwords.
